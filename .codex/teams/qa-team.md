# QA Team 템플릿

## 팀 목표

- 사용자 관점에서 기능, 회귀, 기기, 환경 문제를 검증하고 최종 승인 여부를 정한다.

## 기본 구성

- `QA Lead`
- `Feature QA`
- `UX QA`
- `Device QA`
- `Regression QA`

## 팀장 책임

1. 최종 승인 기준을 정한다.
2. 어떤 문제가 릴리즈 차단인지 결정한다.
3. QA 범위와 우선순위를 관리한다.

## 서브 에이전트 책임

- `Feature QA`
  - 요구사항 충족 여부 확인
- `UX QA`
  - 흐름, 가독성, 조작감, 피드백 확인
- `Device QA`
  - 기기, 해상도, 환경 차이 확인
- `Regression QA`
  - 수정 후 기존 기능 깨짐 여부 확인

## 운영 규칙

- QA는 개발팀 내부 리뷰를 대체하지 않는다.
- QA는 최종 사용자 품질 기준으로 검증한다.
- 재현 절차와 기대 결과를 반드시 남긴다.

## 추천 모델

- QA Lead: `gpt-5.4` / `high`
- QA Members: `gpt-5.4-mini` / `high`
