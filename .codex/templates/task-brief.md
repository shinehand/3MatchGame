# Task Brief

## 작업명

- 예: 튜토리얼 스테이지 1 구현

## 목표

- 이번 작업으로 사용자에게 어떤 가치를 주는지 한 줄로 적는다.

## 범위

- 포함
- 제외

## PM 입력

- 이번 작업의 우선순위
- 담당 팀
- 제외 범위
- 승인 기준

## 기획 입력

- 핵심 루프
- 성공 조건
- 실패 조건
- 시스템/흐름 메모

## 아트 입력

- 필요한 화면
- 필요한 에셋
- UI 제약
- 임시 리소스 허용 여부

## 개발 입력

- 구현 담당
  - 대상 로직/데이터
  - 대상 화면/UI
  - 저장/연동 포인트
  - 내부 리뷰 포인트
  - 테스트 방법

## 완료 조건

- 사용자가 확인할 수 있는 결과
- 개발자가 검증할 수 있는 결과

## QA 입력

- 재현 절차
- 확인 환경/기기
- 반려 기준

## 리스크

- 일정
- 성능
- 아트 의존성
- 기획 미정 요소
- 파일 충돌 가능성
