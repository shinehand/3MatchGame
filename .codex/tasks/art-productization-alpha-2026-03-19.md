# Task Brief

## 작업명

- 제품화 알파: 기획 고도화 + 아트 고도화 + 개발 반영

## 목표

- `Animal Pop Match`를 프로토타입 UI 단계에서 `제품화 알파` 단계로 끌어올린다.

## 범위

- 포함
- 기획 기준선 정리
- 튜토리얼 계층화
- 아트 패키지 정의
- UI 제품화 1차 반영
- 대표 스테이지 QA 게이트 정의
- 제외
- 신규 메타 시스템 대규모 추가
- 과금/광고
- 신규 이벤트 모드
- 대규모 씬 구조 개편

## PM 입력

- 이번 작업의 우선순위
- `아트 고도화 > 기획 기준선 정리 > UI 제품화 > QA 게이트 강화`
- 담당 팀
- PM Team, Planning Team, Art Team, Development Team, QA Team
- 제외 범위
- 신규 대형 기능, 라이브 운영 기능, 과금/광고
- 승인 기준
- 홈에서 첫 플레이 진입까지 설명 없이 이해 가능
- 플레이 화면에서 개발용 UI 비노출
- Stage 1과 밴드 전환 스테이지에 학습 장치 존재
- UI 테마 팩 1차 반영 완료
- 자동 검증과 대표 스테이지 수동 검증 통과

## 기획 입력

- 핵심 루프
- 홈 -> 스테이지 진입 -> 목표 확인 -> 스와이프 매치 -> 클리어/실패 -> 다음 스테이지
- 성공 조건
- 목표, 이동 수, 다음 행동이 첫 시선에서 읽힌다.
- 밴드 전환마다 학습 장치가 들어간다.
- 실패 조건
- 문서 기준선 충돌 지속
- 튜토리얼이 Stage 1에만 머묾
- 시스템/흐름 메모
- 현재 흐름과 목표 흐름을 분리 문서화한다.
- 스테이지 데이터 운영 필드를 잠근다.

## 아트 입력

- 필요한 화면
- 홈, 스테이지 카드, 플레이 HUD, 시작/클리어/실패 오버레이
- 필요한 에셋
- UI 테마 팩
- 보드 상태 배지 세트
- 목표/HUD 세트
- 메타 UI 세트
- 밴드 배경 1-20 세트
- UI 제약
- `block-first`, 동물 얼굴 비가림 금지, 보드 가독성 우선
- 임시 리소스 허용 여부
- 1차 스프린트에서는 임시 리소스 허용, 단 파일명과 연결 규격은 최종 기준으로 고정

## 개발 입력

- 구현 담당
- 대상 로직/데이터
- 튜토리얼 트리거, 설정 화면 흐름, 밴드별 배경/자산 키
- 대상 화면/UI
- 홈, 플레이 HUD, 오버레이, 스테이지 카드
- 저장/연동 포인트
- `stage_id`, 설정 저장, 자산 키 연결
- 내부 리뷰 포인트
- 디버그 UI 잔존 여부, portrait/landscape 동작, 자산 교체 비용
- 테스트 방법
- `./scripts/validate_gameplay.sh`
- 대표 스테이지 수동 플레이

## 완료 조건

- 사용자가 확인할 수 있는 결과
- 더 게임다운 홈/플레이/오버레이 경험
- 대표 스테이지에서 학습 흐름 개선
- 개발자가 검증할 수 있는 결과
- 문서 기준선 동기화
- 자산 패키지 경로와 규격 고정
- 자동 검증 통과

## QA 입력

- 재현 절차
- 홈 -> Stage 1 -> Stage 11 -> Stage 25 -> Stage 50 -> Stage 100
- 확인 환경/기기
- Godot 에디터, headless 검증, Android 실기기
- 반려 기준
- 첫 10초 안에 핵심 정보 미인지
- 개발용 버튼 노출
- HUD가 보드 가독성 침범
- 튜토리얼/오버레이가 다음 행동을 안내하지 못함

## 리스크

- 일정
- 아트 패키지 제작량이 많다.
- 성능
- 고해상도 텍스처와 과한 오버레이 남용 위험
- 아트 의존성
- UI 테마 팩 지연 시 개발 반영 지연
- 기획 미정 요소
- 밴드 튜토리얼 기준, 후반 장애물 확장
- 파일 충돌 가능성
- `main.tscn`, `gameplay.tscn`, `scripts/gameplay.gd`에 집중될 가능성
