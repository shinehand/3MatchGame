# Scene Art Concept

## 1. 목적

- 이 문서는 `Animal Pop Match`의 각 씬을 어떤 아트 컨셉으로 구성할지 정의한다.
- 기준점은 `현재 동물 블록 에셋`이다. 즉, 화면 전체가 동물 블록의 말랑한 스티커형 얼굴 톤과 맞아야 한다.
- 새 컨셉 이미지나 UI 시안은 이 문서의 기준에 맞지 않으면 채택하지 않는다.

## 2. 핵심 톤 정의

### 맞아야 하는 톤

- `동물 스티커`, `말랑한 젤리`, `어린이 그림책`, `라운드 UI`
- 정면 얼굴 중심
- 단순한 실루엣
- 굵고 안정적인 외곽선
- 과하게 화려하지 않은 밝은 파스텔

### 맞지 않는 톤

- 일반 판타지 일러스트 느낌
- 배경이 주인공인 풍경화 톤
- 디테일이 너무 많은 캐릭터 아트
- 사탕이나 쿠키 중심의 디저트 판타지
- 반짝임이 지나치게 많고 UI보다 배경이 튀는 화면

## 3. 전체 화면 공통 원칙

- 모든 씬은 `동물 블록`이 주인공이다.
- 배경은 공간감을 주되, 블록보다 튀지 않아야 한다.
- 캐릭터는 전신보다 얼굴 중심으로 간다.
- 버튼과 패널은 장난감 스티커처럼 보여야 한다.
- 직선형 카드보다 `둥근 코너 + 부드러운 볼륨`이 우선이다.

## 4. 씬별 아트 컨셉

### A. 스플래시/로고 화면

#### 목적

- 게임 첫인상 전달
- `귀여운 동물 퍼즐 게임`이라는 장르 인지

#### 비주얼 방향

- 배경은 하늘색 그라디언트 또는 아주 단순한 초원색
- 동물 2~3마리가 로고 주변에 스티커처럼 배치
- 로고는 부드러운 풍선형 타이포 또는 젤리 간판형

#### 금지 요소

- 복잡한 원근
- 풍경 위주 일러스트
- 보드가 없는 상태에서 과한 장식

### B. 홈 화면

#### 목적

- 유저가 바로 `플레이`를 누르게 만드는 화면

#### 비주얼 방향

- 중앙: 큰 `플레이` 버튼
- 상단 또는 중앙 상부: 게임 로고
- 좌우: 동물 얼굴 3~5종을 포스터처럼 배치
- 하단: 잔디, 구름, 작은 꽃 같은 단순 장식

#### 색감

- 배경: 밝은 하늘색
- 버튼: 노랑 또는 코랄 핑크
- 포인트: 민트, 크림, 복숭아색

#### 핵심 느낌

- `장난감 상자`를 열기 전의 기대감
- 무조건 귀엽고 읽기 쉬워야 함

### C. Stage 시작 오버레이

#### 목적

- 이번 스테이지 목표와 이동 수를 빠르게 이해

#### 비주얼 방향

- 중앙 팝업 카드
- 상단 제목 배지: `Stage 1`, `Easy`
- 본문: 목표 아이콘과 숫자
- 하단: 큰 `시작` 버튼

#### 재질감

- 종이 카드보다 `쿠션형 스티커 보드`
- 목표 칩은 동물 얼굴 배지 형태

### D. 플레이 화면

#### 목적

- 보드 가독성 최우선
- HUD는 단순하고 강하게

#### 구성 원칙

- 보드 뒤 배경은 흐린 하늘/초원 패턴 수준으로 유지
- 보드 판은 하늘색 또는 민트색의 말랑한 보드판
- 보드 셀은 너무 진하지 않게, 동물 얼굴이 가장 잘 보이게
- 우측 HUD는 장난감 카드 더미처럼 보이게 구성

#### 톤 규칙

- 실제 귀여운 포인트는 `블록`과 `목표 칩`에서 나온다.
- 배경은 귀엽지만 조용해야 한다.
- 화면 전체가 “예쁜 일러스트”보다 “플레이 가능한 장난감”처럼 보여야 한다.

### E. 특수 블록 연출

#### 목적

- 큰 보상 감각 전달

#### 비주얼 방향

- 줄 제거 특수 블록:
  - 동물 얼굴 위에 작은 리본형 방향 배지
  - `↔`, `↕` 같은 심벌은 보조 요소로만 사용

- 폭발 블록:
  - 얼굴 주변에 별모양 프레임 또는 왕관형 테두리
  - 배경보다 얼굴이 먼저 읽혀야 함

#### 금지 요소

- 기호가 얼굴보다 커지는 구성
- 금속적, 하드한 이펙트

### F. 장애물 씬 요소

#### 현재 장애물: 덤불

- 덤불은 자연 오브젝트이지만 현실적인 수풀처럼 보이면 안 된다.
- `둥근 잎이 겹친 스티커`, `솜사탕 같은 부시` 느낌으로 간다.

#### 역할

- 위협적인 오브젝트가 아니라 `정리해야 하는 귀여운 방해물`

### G. 클리어 오버레이

#### 목적

- 성취감과 다음 플레이 유도

#### 비주얼 방향

- 큰 별 1~3개
- 웃는 동물 얼굴 1종
- 다음 스테이지 버튼은 가장 크고 밝게

#### 느낌

- 축하 카드
- 어린이 스티커 북 보상 페이지

### H. 실패 오버레이

#### 목적

- 좌절보다 재도전 유도

#### 비주얼 방향

- 살짝 아쉬운 표정의 동물
- 너무 어두워지지 않는 채도
- `재도전` 버튼이 명확하게 보여야 함

#### 금지 요소

- 회색/검정 위주의 실패 연출
- 과한 패배감

## 5. 각 씬의 색 사용 원칙

### 홈 화면

- 하늘색 + 노랑 + 코랄 핑크

### 플레이 화면

- 하늘색 + 민트 + 크림

### 클리어 화면

- 크림 + 노랑 + 부드러운 핑크

### 실패 화면

- 하늘색 유지 + 살짝 다운된 복숭아색

## 6. 화면 간 연결성

- 홈 화면 캐릭터와 블록 캐릭터가 같은 얼굴형이어야 한다.
- 홈 화면에서 본 동물과 플레이 화면 목표 칩의 표정 언어가 같아야 한다.
- 클리어/실패 오버레이의 동물 표정도 동일 세계관 안에 있어야 한다.

## 7. 실제 제작 순서

1. 동물 블록 최종 얼굴형 고정
2. 홈 화면 키 비주얼 제작
3. 플레이 화면 배경과 HUD 카드 톤 제작
4. 특수 블록 배지와 장애물 톤 제작
5. 클리어/실패 오버레이 감정 표현 제작

## 8. 다음 이미지 생성 기준

- 프롬프트에 반드시 `sticker-like animal faces`, `rounded casual mobile game UI`, `quiet pastel background`를 포함한다.
- `풍경화`, `storybook landscape`, `detailed illustration` 쪽으로 프롬프트가 새면 폐기한다.
- 배경 이미지를 만들 때도 항상 `board readability first`를 넣는다.
- 단, 가능하면 생성형 풀씬 이미지보다 [block-first-scene-design.md](/Users/shinehandmac/Documents/puzzle/docs/art/block-first-scene-design.md) 기준의 레이아웃 보드 작업을 우선한다.
