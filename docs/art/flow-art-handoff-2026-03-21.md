# Flow Art Handoff 2026-03-21

## 목적

- 상세 기획안 기준으로 아트팀이 화면 단위로 필요한 자산을 바로 생산할 수 있게 한다.
- 우선순위는 `메타 상태 차별화 -> 결과 보상 -> 플레이 순간 피드백` 순으로 둔다.

## 화면별 아트 요구사항

### 홈

- 메인 CTA는 `이어하기` 역할이 즉시 보이도록 유지한다.
- 서브 CTA인 `스테이지`, `설정`은 메인 CTA보다 한 단계 가볍게 보이게 한다.
- 홈 마스코트는 최소 3종 체계가 필요하다.
- 브랜드 대표형
- 진행 유도형
- 이벤트/공지형

### 스테이지 선택

- 스테이지 선택은 이제 홈 오버레이가 아니라 `전용 스테이지 라인 씬` 기준으로 본다.
- 씬에는 `밴드 스토리`, `현재 추천 스테이지`, `전체 카드 그리드`가 함께 들어간다.
- 카드 상태 세트가 최우선이다.
- `도전 가능`
- `현재 진행`
- `클리어`
- `3스타 강조`
- `피날레`
- 현재는 배지 중심이라 카드 프레임 상태 차이가 약하다.
- 다음 라운드는 카드 프레임 자체의 상태 차이를 만든다.

### 설정

- 설정은 기능 수가 적으므로 오버레이 프레임의 안정감과 토글 상태 구분이 중요하다.
- `ON/OFF` 상태는 색과 눌림감 둘 다로 구분한다.

### 시작 오버레이

- 튜토리얼 스테이지 전용 시각 위계가 필요하다.
- 일반 시작 팝업과 같은 프레임을 쓰더라도 `튜토리얼 리본`, `학습 배지`, `주의 컬러 띠` 중 최소 1개를 더 둔다.

### 플레이 중

- 순간 피드백 FX 세트가 필요하다.
- 선택 글로우
- 잘못된 이동
- 블로커 타격
- 목표 달성 직전
- 목표 완료 펄스

### 결과 오버레이

- 현재는 성공/실패 표정 분기가 있으나 보상감이 약하다.
- 우선 생산 자산은 아래다.
- 클리어 스탬프
- 별 획득 배너
- 실패 소프트 피드백 카드
- 전체 완료 타이틀 카드

## 이번 라운드 우선순위

### Priority 1

- 스테이지 카드 상태 세트
- 이유: 메타 진행감을 가장 빠르게 끌어올린다.
- 반영 상태
- `stage_card_frame_current`
- `stage_card_frame_cleared`
- `stage_card_frame_finale`
- `stage_card_frame_locked`

### Priority 2

- 결과 오버레이 보상 세트
- 이유: 시작부터 종료까지 플로우에서 종료 경험이 가장 약하다.

### Priority 3

- 플레이 순간 피드백 FX 세트
- 이유: 게임 감각과 HUD 상태 전달을 직접 강화한다.

## 제작 방식

- UI 프레임, 배지, 상태 아이콘은 SVG 우선
- 표정형 마스코트와 결과 일러스트는 Local SD 기반 초안 후 수동 선별
- OpenAI Image API는 고난도 편집이나 일관성 보정 시 보조로 사용

## 추가 반영 상태

- 스테이지 라인 씬용 스토리 노드 세트 반영
- 중후반 배경 세트 반영
- `forest_edge_1`
- `garden_1`
- `garden_2`
- `night_shade_1`
- `skyline_1`
- `skyline_2`
- `finale_1`

## 승인 기준

- 256px 이하 축소에서도 상태 차이를 읽을 수 있어야 한다.
- 배경은 보드보다 앞에 나오면 안 된다.
- 상태는 텍스트 없이도 최소 1차 구분이 가능해야 한다.
- 튜토리얼, 클리어, 실패, 피날레는 서로 같은 감정 톤으로 보이면 반려한다.
