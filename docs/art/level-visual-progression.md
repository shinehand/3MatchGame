# Level Visual Progression

## 목적

- 이 문서는 `Animal Pop Match`의 100개 스테이지가 같은 세계관 안에서 점진적으로 새로워 보이도록 하는 시각 규칙을 정의한다.
- 기준점은 현재 동물 블록 에셋과 `block-first` 씬 언어다.
- 핵심 목표는 `새로운 그림을 계속 늘리는 것`이 아니라 `같은 자산을 조합해 난이도와 분위기만 확장하는 것`이다.

## 비협상 기준

- 동물 블록이 항상 화면의 주인공이어야 한다.
- 배경은 읽기 쉬워야 하고 보드보다 튀면 안 된다.
- UI는 모바일에서 즉시 구분되어야 하며, 과한 장식보다 명확함이 우선이다.
- 전신 캐릭터 포스터형 일러스트는 사용하지 않는다.
- 사탕, 쿠키, 보석 같은 다른 캐주얼 테마로 톤이 이동하지 않는다.
- 새 스테이지를 만들더라도 얼굴형, 외곽선 두께, 파스텔 채도는 현재 블록 스타일에서 벗어나지 않는다.

## 100스테이지 시각 구조

### 기본 원칙

- 100개 스테이지는 10개 단위 밴드로 나눈다.
- 한 밴드 안에서는 배경 톤과 HUD 강도를 거의 고정하고, 보드 모양과 장애물 밀도로만 변화시킨다.
- 밴드가 바뀔 때만 `새 테마가 들어왔다`는 느낌을 주고, 스테이지 하나마다 미술을 새로 만든 느낌은 피한다.
- 신규 시각 자산은 밴드 단위로만 추가한다.

### 밴드 규칙

- 1-10: 튜토리얼과 적응 구간
- 11-20: 밝은 초원 구간
- 21-30: 잔잔한 숲 가장자리 구간
- 31-40: 연못과 물가 구간
- 41-50: 꽃밭과 정원 구간
- 51-60: 둥근 언덕과 돌길 구간
- 61-70: 나무 그늘과 밤하늘 구간
- 71-80: 구름, 바람, 공중감이 있는 구간
- 81-90: 복합 목표와 고밀도 장애물 구간
- 91-100: 최종 도전과 보상 연출 강화 구간

## 스테이지 밴드별 시각 방향

### 1-10

- 목적: 조작과 기본 규칙 이해
- 배경: 가장 밝고 단순한 하늘색, 초원색, 구름 몇 개
- 보드 모양: 거의 정사각형 유지, 가장자리만 약간 깎는 정도
- 장애물: 없음 또는 1종만 아주 적게 사용
- 연출: 클리어/실패 팝업은 가장 단순한 버전 사용

### 11-20

- 목적: 목표 칩과 특수 블록 인지
- 배경: 초원에 작은 꽃과 부드러운 곡선 추가
- 보드 모양: 완만한 곡선형 또는 둥근 모서리 컷아웃
- 장애물: 덤불을 얇게 배치
- 연출: 4매치 줄 제거 배지와 매치 팝 이펙트 강조

### 21-30

- 목적: 보드 모양 변화 적응
- 배경: 숲 가장자리, 나무 실루엣은 멀리 배치
- 보드 모양: 가운데가 넓고 위아래가 좁은 형태를 도입
- 장애물: 덤불이 군집으로 나오기 시작
- 연출: 낙하와 리필이 더 잘 보이도록 배경 채도 소폭 낮춤

### 31-40

- 목적: 연쇄와 특수 블록 조합 학습
- 배경: 연못, 물방울, 작은 갈대 같은 저채도 요소
- 보드 모양: 좌우 비대칭이나 절반 컷 형태 사용
- 장애물: 덤불 외에 같은 톤의 꽃덩어리 변형 가능
- 연출: 5매치 폭발 블록의 별 프레임을 더 선명하게 사용

### 41-50

- 목적: 점수와 목표 동시 관리
- 배경: 꽃밭, 부드러운 원형 패턴, 낮은 대비의 장식
- 보드 모양: 원형에 가까운 중앙 보드, 외곽 빈칸 허용
- 장애물: 덤불 밀도를 높이고 구역 분할을 시작
- 연출: 목표 달성 칩의 진행 표시를 더 두드러지게 표현

### 51-60

- 목적: 중반 피로를 줄이면서 변주 유지
- 배경: 언덕, 돌길, 나무 그늘 같은 안정감 있는 구성
- 보드 모양: 계단형 또는 둔각형 컷아웃
- 장애물: 덤불과 낮은 장벽형 오브젝트를 조합 가능
- 연출: 클리어 보상 팝업에 별, 리본, 동물 표정 조합 추가

### 61-70

- 목적: 숙련자용 템포 증가
- 배경: 밤하늘을 아주 약하게 도입하되 밝기는 유지
- 보드 모양: 가늘어진 통로형, 중앙 압축형 등 난도 있는 형태
- 장애물: 목표 달성 동선을 막는 패턴형 배치
- 연출: 특수 블록 발동 시 대비를 높이고, 이펙트 길이는 짧게 유지

### 71-80

- 목적: 구도 읽기와 계획형 플레이 강화
- 배경: 바람, 구름, 떠 있는 느낌의 장식으로 가벼운 변화
- 보드 모양: 좌우 분절형, 상하 분절형, 중간 브릿지형
- 장애물: 덤불을 구획 사이에 넣어 경로를 고민하게 함
- 연출: 목표 칩과 보드의 시선 흐름을 더 강하게 분리

### 81-90

- 목적: 복합 목표와 고밀도 퍼즐
- 배경: 현재 테마를 재사용하되 색 대비만 약간 강화
- 보드 모양: 비정형이지만 지나치게 복잡하지 않은 수준 유지
- 장애물: 복수 종류의 방해 요소를 소량 결합
- 연출: 성공과 실패 팝업의 감정 표현을 조금 더 크게

### 91-100

- 목적: 최종 구간의 집중감과 마무리감
- 배경: 기존 테마를 섞되 과한 새 장식은 금지
- 보드 모양: 압축형, 분리형, 교차형 등 하이브리드 형태
- 장애물: 높은 밀도와 제한된 낙하 공간을 사용
- 연출: 클리어 오버레이와 별 연출을 가장 풍부하게 사용

## 배경 회전 전략

- 배경은 100개를 모두 새로 만들지 않는다.
- 한 밴드당 1개의 메인 배경과 1개의 서브 변형만 둔다.
- 실제 사용은 `메인`, `좌우 반전`, `색감 살짝 조정`, `장식 위치 약간 이동` 정도로만 변형한다.
- 배경은 보드 뒤에서 조용히 있어야 하며, 스테이지 성격은 주로 보드 모양과 장애물로 전달한다.
- 배경 교체 시점은 밴드 시작 스테이지 또는 10의 배수 스테이지로만 둔다.

### 배경 자산 운영

- 초반 밴드 2개는 기존 초원 배경 계열 재사용
- 중반 밴드는 동일 구조에 요소만 교체
- 후반 밴드는 어두운 느낌보다 대비 조절 중심
- 모든 배경은 `board readability first` 조건을 통과해야 한다.

## HUD / 팝업 변형 한계

### HUD

- HUD 레이아웃은 100스테이지 동안 거의 고정한다.
- 바꿔도 되는 것은 색 포인트, 아이콘 강조, 진행바 길이 정도다.
- HUD 구조를 매 스테이지 바꾸면 학습 비용이 커지므로 금지한다.

### 팝업

- 시작, 클리어, 실패 팝업은 공통 프레임을 사용한다.
- 밴드별로 바꿀 수 있는 것은 제목 배지, 배경색, 작은 장식 1개 정도다.
- CTA 버튼 위치는 절대 바꾸지 않는다.
- 텍스트 길이는 모바일 한 화면에 들어오는 수준으로 제한한다.

### Variation budget

- 홈 화면: 2세트
- Stage start: 2세트
- Clear popup: 2세트
- Fail popup: 2세트
- Goal chip: 1세트 기본, 목표 종류별로 아이콘만 교체

## 장애물 및 이펙트 점진 규칙

### 장애물

- 1단계: 없음
- 2단계: 단일 덤불
- 3단계: 군집 덤불
- 4단계: 덤불 + 구역 분할
- 5단계: 덤불 + 높은 밀도 구간

### 이펙트

- 초반: 작은 팝, 짧은 별 버스트
- 중반: 특수 블록 배지, 낙하 효과, 약한 흔들림
- 후반: 보드 전체에 영향이 있는 경우만 이펙트를 확장

### 금지

- 번쩍이는 네온형 효과
- 금속성 폭발
- 화면 전체를 덮는 과한 파티클

## 재사용 에셋 목록

### 필수 재사용 자산

- 동물 블록 5종 얼굴 에셋
- 덤불 장애물 에셋
- 매치 버스트 에셋
- 배경 하늘/초원 계열 1세트
- HUD 카드 프레임
- 목표 칩 프레임
- 특수 블록 배지 아이콘
- 시작/클리어/실패 공통 팝업 프레임

### 추가 가능 자산

- 밴드 전환용 배경 장식 1개씩
- 클리어 별 연출 1종
- 실패용 아쉬운 표정 마스코트 1종

## Godot 통합 원칙

- 배경, HUD, 팝업, 이펙트는 `res://assets/` 아래에 기능별로 분리한다.
- 스테이지 밴드별 배경 키는 `stage_band` 같은 단순 키로 관리한다.
- 특수 블록과 장애물의 시각 차이는 텍스처 교체보다 배지, 프레임, 오버레이 조합으로 만든다.
- 새 스테이지가 늘어나도 씬 구조는 바꾸지 않고 데이터와 텍스처만 추가한다.
- 보드 가독성을 해치면 아트가 아니라 자산 강도를 먼저 낮춘다.

## 생산 기준

1. 새 밴드를 추가할 때는 메인 배경 1개, 보조 배경 1개, 팝업 변형 1개만 만든다.
2. 스테이지 10개마다 아트 리뷰를 한다.
3. 100스테이지 전체에서 가장 중요한 것은 자산 수가 아니라 읽기 쉬운 일관성이다.
4. 어떤 새 자산도 현재 동물 블록 얼굴 톤보다 앞서면 채택하지 않는다.
