# Block-First Scene Design

## 1. 왜 기존 컨셉이 틀렸는가

- 현재 게임의 실제 핵심 비주얼은 `동물 얼굴 블록`이다.
- 그런데 지금까지 생성한 씬 컨셉은 `일반 캐주얼 게임 일러스트` 쪽으로 흘렀다.
- 즉, 화면 전체가 실제 게임 에셋을 설명하지 못했다.

## 2. 앞으로의 기준

- 씬 컨셉은 `블록 에셋에서 출발`해야 한다.
- 배경, 버튼, 오버레이는 블록을 돋보이게 하는 보조 요소다.
- 씬 아트는 “예쁜 배경 그림”이 아니라 “블록이 살 수 있는 무대”를 만드는 작업이다.

## 3. 씬 설계 출발점

### 가장 먼저 고정할 것

- 동물 얼굴의 비율
- 눈/입/귀의 단순한 표정 언어
- 타일 외곽선 두께
- 하이라이트와 그림자 방식

### 그 다음 고정할 것

- 보드 판의 재질감
- 목표 칩의 카드 형태
- 버튼의 둥근 장난감 같은 볼륨

## 4. 씬별 실제 기준

### 홈 화면

- 동물 블록 캐릭터를 `타일 바깥으로 확장한 얼굴 마스코트`처럼 사용
- 전신 일러스트 금지
- 로고, 플레이 버튼, 동물 얼굴 3개 정도만으로 구성

### 플레이 화면

- 보드가 화면의 주인공
- 배경은 거의 단색에 가까운 부드러운 패턴
- HUD는 블록과 같은 세계관의 둥근 카드
- 목표 칩은 실제 블록 얼굴을 그대로 축소해서 사용

### 오버레이

- 결과 팝업도 블록과 같은 표정 언어 사용
- 클리어는 웃는 얼굴, 실패는 살짝 아쉬운 얼굴
- 별, 리본, 배지 같은 단순 보상 장식만 사용

## 5. 하지 말아야 할 것

- 풍경 중심 컨셉 아트
- 전신 캐릭터 포스터
- 배경이 블록보다 눈에 띄는 구성
- 블록과 전혀 다른 렌더 스타일
- 과한 브러시 질감, 회화적 톤

## 6. 올바른 다음 산출물

다음 아트 산출물은 아래 순서로 만든다.

1. `홈 화면 레이아웃 보드`
2. `플레이 화면 레이아웃 보드`
3. `오버레이 UI 보드`
4. `블록 스타일에 맞춘 홈용 얼굴 마스코트`

## 7. 작업 방식

- 새로운 씬 시안은 가능하면 `실제 블록 에셋`을 써서 만든다.
- 생성형 이미지가 필요하더라도 `블록과 동일한 얼굴형`, `스티커형`, `조용한 배경`을 강제한다.
- 생성 이미지가 블록과 다르면 폐기한다.
