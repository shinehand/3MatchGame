# Current Version Audit 2026-03-18

## 목적

- 현재 구현이 기획 의도에 얼마나 맞는지 확인한다.
- 다음 구현 우선순위를 명확히 정한다.

## 총평

- 현재 버전은 `매치3 프로토타입`을 넘어 `100스테이지 캠페인 골격`까지는 올라왔다.
- 다만 `디버그 성격 UI 노출`, `학습 설계 부족`, `플레이 화면 정보 과밀`, `아트/리소스 미완성` 때문에 아직 `정식 게임처럼 느껴지는 상태`는 아니다.

## 기획 적합성 요약

### 맞게 구현된 부분

- 홈 -> 스테이지 시작 오버레이 -> 플레이 -> 클리어/실패 루프가 존재한다.
- 100스테이지 카탈로그, 저장 구조, 비정형 보드, 목표/점수/덤불/특수 블록이 데이터 기반으로 연결돼 있다.
- 유효/무효 스왑, 낙하, 리필, 연쇄, 목표 칩, 사운드/햅틱까지 코어 손맛 구조는 살아 있다.

### 의도와 어긋난 부분

- 플레이어가 언제든 누를 수 있는 `셔플` 버튼이 퍼즐 경제를 깨고 있다.
- `디버그 매치` 버튼이 실제 플레이 HUD에 남아 있다.
- Stage 1 외에는 밴드 전환용 소프트 튜토리얼이 사실상 없다.
- 플레이 화면 필수 정보로 정의된 `일시정지 버튼`이 없다.
- 첫 진입 UX가 텍스트 중심이라 `첫 10초 안에 이해` 목표와 거리가 있다.
- 기획 문서 일부가 100스테이지 현재 상태와 어긋나 기준선이 흔들린다.

## 아트/UX 적합성 요약

### 맞는 방향

- 보드가 화면의 주인공이어야 한다는 큰 방향은 유지되고 있다.
- 동물 얼굴 블록 중심 세계관은 맞다.
- 홈 화면도 전신 캐릭터보다 블록 얼굴을 쓰려는 방향은 맞다.

### 부족한 부분

- 버튼, 패널, 스테이지 카드가 거의 기본 위젯 상태다.
- 플레이 화면은 정보 위계가 흐리고 개발용 컨트롤이 많이 보인다.
- 타일 선택 상태와 특수 블록 표시는 얼굴을 가리거나 세계관과 어긋난다.
- 홈 화면과 스테이지 선택은 아직 `block-first` 아트 기준에 못 미친다.
- 색감 계층과 배경 불투명도 때문에 전체가 부옇게 보일 위험이 있다.
- UI 프레임, 버튼 상태, 특수 배지, 감정형 오버레이 등 필수 2D 리소스가 비어 있다.

## 결론

- 현재 구현 수준: `게임 코어 + 캠페인 데이터 + 프로토타입 UI`
- 아직 부족한 수준: `정식 UX`, `일관된 시각 언어`, `학습 설계`, `메타 완성도`

## 다음 우선순위

1. `셔플`, `디버그 매치`를 일반 플레이 UI에서 제거하거나 개발 모드로 숨긴다.
2. 플레이 씬에 `일시정지/설정`을 추가한다.
3. `Stage 11`, `25`, `45`, `65`, `85`, `95` 같은 밴드 전환 지점에 소프트 튜토리얼을 넣는다.
4. `UI 테마 팩 + 스테이지 카드 + 특수 배지 + 감정형 오버레이`를 아트 우선 작업으로 진행한다.
5. `game-design.md`, `implementation-review-2026-03-18.md`를 현재 상태로 갱신한다.
