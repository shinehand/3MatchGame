# Game Flow Detailed 2026-03-21

## 목적

- 현재 구현 기준으로 `앱 실행 -> 홈 -> 스테이지 선택/설정 -> 인게임 시작 -> 플레이 -> 클리어/실패 -> 다음 행동` 흐름을 하나의 문서로 잠근다.
- 아트팀과 개발팀이 같은 화면 단위, 같은 버튼 의미, 같은 승인 기준으로 움직이게 만든다.

## 이번 기준선

- 알파 기준에서는 `스플래시`를 두지 않는다. 앱 실행 시 바로 홈으로 진입한다.
- 홈 메인 CTA는 항상 `시작`으로 사용한다.
- `시작`은 곧바로 보드 진입이 아니라 `전용 스테이지 선택 씬`으로 이동하는 의미다.
- `스테이지 선택`은 숨김 기능이 아니라 노출 기능으로 본다.
- `설정`은 홈에서 접근 가능한 기능으로 본다.
- 홈으로 복귀하면 `현재 선택 스테이지가 강조된 스테이지 선택 씬으로 다시 진입`하는 구조로 이어진다.
- 이번 라운드에서는 `중간 턴 저장`은 하지 않는다.

## 시작부터 종료까지의 기준 플로우

### 1. 앱 실행

- 저장 데이터를 읽는다.
- 바로 홈 화면을 보여준다.
- 첫 세션이면 기본 선택 스테이지는 `Stage 1`이다.
- 재방문이면 마지막 선택 스테이지를 홈 진행 정보와 스테이지 선택 씬 강조 상태에 반영한다.

### 2. 홈 화면

- 가장 크게 보이는 정보는 `구조 작전 시작` CTA와 `다음 추천 스테이지`다.
- 메인 버튼은 항상 `시작`을 사용한다.
- `시작`을 누르면 현재 추천 스테이지가 강조된 스테이지 선택 씬으로 이동한다.
- 서브 버튼은 항상 노출한다.
- `스테이지 라인`: 전용 스테이지 선택 씬 진입
- `설정`: 사운드/햅틱 설정 오버레이 진입
- 홈 정보 카드에는 다음 정보를 노출한다.
- 다음 추천 스테이지
- 해금 수
- 클리어 수
- 누적 별 수
- 사운드/햅틱 현재 값

### 3. 스테이지 선택

- 스테이지 선택은 홈 위 오버레이가 아니라 `전용 씬`으로 운영한다.
- 씬은 `밴드 스토리 라인`, `현재 추천 스테이지`, `전체 스테이지 카드 그리드`를 함께 보여준다.
- 해금된 스테이지는 탭 시 바로 선택되고 인게임으로 진입한다.
- 잠긴 스테이지는 진입할 수 없다.
- 카드 상태는 최소 아래를 구분해야 한다.
- 잠김
- 도전 가능
- 현재 진행
- 밴드 피날레
- 누적 별 상태
- 스테이지 선택 씬 상단에는 전체 진행 요약을 유지한다.

### 4. 설정

- 홈에서만 접근한다.
- 알파 기준 설정 항목은 아래 두 개로 제한한다.
- 사운드 on/off
- 햅틱 on/off
- 설정 값은 저장되고 다음 플레이에도 유지된다.
- 이번 라운드에서 BGM, 언어, 계정, 푸시는 범위에서 제외한다.

### 5. 스테이지 시작 오버레이

- 스테이지 진입 직후 무조건 한 번 노출한다.
- 아래 다섯 요소를 항상 포함한다.
- 스테이지명
- 난이도
- 목표 요약
- 남은 이동 수
- 튜토리얼 문구 또는 핵심 팁
- 튜토리얼 스테이지에서는 `학습 포인트`가 일반 스테이지보다 더 분명하게 보이게 한다.
- CTA는 `시작` 하나만 둔다.

### 6. 플레이 중

- HUD에서 최소 아래 정보는 항상 읽혀야 한다.
- 목표 칩
- 남은 이동 수
- 현재 스테이지
- 플레이 중 버튼은 아래 의미를 유지한다.
- `재시작`: 현재 스테이지 다시 시작
- `다음 스테이지`: 클리어 이후에만 실제 진행
- `홈으로`: 홈으로 복귀
- 튜토리얼 스테이지는 오버레이와 HUD 중 최소 한 곳에서 학습 포인트를 유지한다.

### 7. 일시정지

- 현재 기준 액션은 두 개다.
- `계속`
- `홈으로`
- `홈으로`를 선택하면 현재 선택 스테이지를 저장한 채 홈으로 이동한다.
- 다음 진입은 `해당 스테이지 처음부터 다시 시작`이다.

### 8. 클리어

- 클리어 오버레이에는 아래 정보를 포함한다.
- 스테이지 클리어 상태
- 별 획득 결과
- 다음 행동 2개 이하
- 기본 액션은 아래를 사용한다.
- 일반 스테이지: `다음 스테이지`, `홈으로`
- 마지막 스테이지: `홈으로`, `다시 플레이`
- 이번 라운드에서는 별 세부 집계 화면을 별도 분리하지 않는다.

### 9. 실패

- 실패 오버레이는 부정적 피드백보다 `재도전 유도`를 우선한다.
- 기본 액션은 `재도전` 하나로 유지한다.
- 실패 힌트는 목표와 남은 부족 요소를 짧게 안내한다.

## 튜토리얼 기준선

- Stage 1은 단계형 소프트 튜토리얼을 유지한다.
- 나머지 학습 스테이지는 `짧은 문구 기반 소프트 튜토리얼`을 사용한다.
- 현재 운영 기준 학습 스테이지는 아래다.
- `1, 3, 7, 11, 15, 25, 45, 65, 85, 95`
- 이번 라운드에서는 `튜토리얼 다시 보지 않기` 저장까지는 하지 않는다.

## 홈 복귀와 이어하기 정의

- 홈의 `시작`은 `마지막으로 선택한 스테이지가 강조된 스테이지 선택 씬으로 이동`하는 의미다.
- 실제 플레이 시작은 스테이지 선택 씬에서 카드를 고른 뒤 이뤄진다.
- `중간 턴, 현재 보드, 남은 이동 수`를 복원하는 방식은 아니다.
- 이 정의는 홈 문구, QA 체크리스트, 개발 저장 구조에서 동일해야 한다.

## 아트 승인 기준

- 홈은 `시작`과 `스테이지 라인`의 역할 차이가 즉시 보인다.
- 스테이지 선택 씬은 카드 상태만 보고 `잠김/현재/클리어/피날레`를 구분할 수 있다.
- 시작 오버레이는 튜토리얼 스테이지와 일반 스테이지의 위계가 다르게 보인다.
- 클리어와 실패는 표정, 리본, 보상 배지에서 즉시 구분된다.
- 플레이 중 선택, 잘못된 입력, 목표 완료 직전, 목표 완료 순간의 피드백 차이가 보여야 한다.

## 개발 승인 기준

- 첫 세션은 홈에서 `시작`을 누른 뒤 Stage 1이 강조된 스테이지 선택 씬에 진입한다.
- 재방문 세션은 홈에서 추천 Stage 번호가 노출되고, 스테이지 선택 씬에서 해당 스테이지가 강조된다.
- 스테이지 라인과 설정 버튼은 홈에서 접근 가능하다.
- 설정 변경은 씬 전환 후에도 유지된다.
- 시작/일시정지/클리어/실패 오버레이의 버튼 의미가 상태마다 바뀌지 않는다.
- 홈 복귀 후 다시 시작했을 때 저장 규칙이 문서와 동일하다.
