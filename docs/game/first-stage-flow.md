# 첫 화면부터 첫 스테이지 진입까지

## 1. 목적

- 이 문서는 `앱 실행 직후`부터 `Stage 1 플레이 시작`까지의 화면 흐름과 UX 의도를 정의한다.
- 목표는 첫 세션 유저가 설명 없이도 `플레이 버튼 -> 목표 확인 -> 첫 매치`까지 자연스럽게 들어가게 만드는 것이다.

## 2. 전체 흐름

1. 스플래시 또는 로고 화면
2. 홈 화면
3. 플레이 버튼 선택
4. Stage 1 시작 오버레이
5. 실제 플레이 시작

## 3. 스플래시/로고 화면

### 목적

- 게임의 무드와 브랜드를 가장 짧게 전달
- 너무 길지 않게 1~2초 안에 홈 화면으로 이동

### 표시 요소

- 게임 로고 `Animal Pop Match`
- 귀여운 동물 블록 2~3종
- 부드러운 배경

### UX 원칙

- 강한 인터랙션은 넣지 않는다.
- 로고는 읽기 쉬워야 한다.
- 로딩이 없는 상태라면 빠르게 홈 화면으로 넘긴다.

## 4. 홈 화면

### 목적

- 유저가 무엇을 눌러야 하는지 고민하지 않게 한다.
- 현재 저장 진행도가 없으면 `Stage 1` 시작이 바로 보이게 한다.
- 게임의 목표가 `동물 구출`이라는 것을 첫 화면에서 짧게 이해시킨다.

### 우선 정보

- 게임 타이틀
- 대표 동물 비주얼
- 메인 버튼 `플레이`
- 서브 버튼 `설정`
- 향후 확장 영역: `스테이지 선택`, `이벤트`, `진행도`
- 짧은 구조 카피
  - 예시: `밀렵꾼에게 잡힌 동물들을 구해 주세요`

### 메인 버튼 규칙

- 첫 세션: `플레이`를 누르면 Stage 1 시작
- 재방문 세션: 최근 플레이 스테이지 재개 또는 스테이지 선택으로 이동

### 홈 화면 의도

- 첫 화면에서 정보량을 줄인다.
- 목표 설명을 홈에 길게 쓰지 않는다.
- 게임 시작 CTA를 가장 크게 둔다.
- 단, `왜 플레이하는지`는 한 줄로 분명하게 전달한다.

## 5. Stage 1 시작 오버레이

### 목적

- 첫 플레이 전에 `이번 판 목표`와 `이동 수`를 짧게 인지시킨다.
- 튜토리얼 문구를 너무 길지 않게 전달한다.

### 표시 요소

- 제목: `Stage 1`
- 난이도: `Easy`
- 목표: `토끼 10개 수집`
- 이동 수: `12`
- 짧은 튜토리얼 문구
- 짧은 구조 맥락
  - 예시: `정글 입구의 토끼들이 우리에 갇혀 있습니다.`
- 버튼: `시작`

### 문구 원칙

- 긴 문단보다 짧은 문장 1~2줄
- 예시:
  - `토끼를 10개 구해 주세요.`
  - `블록을 스와이프해 3개 이상 맞추면 됩니다.`

## 6. 첫 플레이 진입 직후

### Stage 1에서 반드시 느껴야 할 것

- 블록이 무엇인지 바로 보인다.
- 어떤 동물을 모아야 하는지 우측 목표 칩에서 바로 읽힌다.
- 잘못된 스와이프도 즉시 피드백을 준다.
- 첫 유효 매치 후 `낙하 -> 목표 진행`이 선명하게 보인다.
- 첫 스테이지가 `첫 구조 임무`처럼 느껴진다.

### 첫 스테이지 UX 요구사항

- 토끼 목표 칩이 가장 먼저 눈에 들어온다.
- 남은 이동 수가 목표 다음으로 잘 보인다.
- 시작 직후 사용법은 `짧은 고정 팁`으로만 보여준다.
- Stage 1에서는 장애물과 복잡한 목표를 넣지 않는다.

## 7. 첫 세션 튜토리얼 원칙

### 하드 튜토리얼 대신 소프트 튜토리얼

- 가능한 스왑 하나를 강제로 요구하지 않는다.
- 대신 목표 칩, 시작 오버레이, 짧은 팁 문구로 방향만 준다.

### 왜 이렇게 가는가

- 매치3 장르는 진입 규칙이 익숙한 유저가 많다.
- 강제 튜토리얼은 반복 플레이에서 피로도가 높다.
- 첫 버전에서는 빠른 진입이 더 중요하다.

## 8. 실패/예외 처리

### 홈 화면에서

- 저장 데이터가 없으면 무조건 Stage 1로 보낸다.
- 저장 데이터가 깨졌으면 Stage 1로 리셋한다.

### Stage 시작 시

- 스테이지 데이터가 없으면 기본 Stage 1을 불러온다.
- 목표가 비어 있으면 진입하지 않고 디버그 로그를 남긴다.

## 9. 개발 관점 완료 조건

- 앱 실행 후 3탭 이내에 Stage 1 진입
- 첫 화면에서 메인 CTA가 명확
- Stage 1 진입 전 목표와 이동 수를 확인 가능
- Stage 1 시작 직후 첫 매치까지 혼란이 없음

## 10. 후속 확장

- 홈 화면에 최근 진행 스테이지 카드 추가
- 스테이지 선택 버튼 분리
- 누적 별 개수 표시
- 이벤트 배너와 기간 한정 모드
