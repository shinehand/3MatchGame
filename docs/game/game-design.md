# Animal Pop Match 기획서

## 1. 게임 개요

- 장르: 스테이지 기반 매치3 퍼즐
- 플랫폼: 모바일 Android 우선
- 화면 방향: landscape 고정
- 조작: 터치 후 인접 블록 방향으로 스와이프
- 목표 유저: 가볍게 즐길 수 있는 캐주얼 퍼즐 유저

## 2. 핵심 판타지

- 밀렵꾼에게 붙잡힌 동물 친구들을 구출하는 구조 판타지
- 한 번의 좋은 스왑이 연쇄 반응으로 이어지는 만족감
- 동물 캐릭터의 표정, 사운드, 반응으로 `구출 성공`의 보상 강화

## 3. 세계관과 스토리 축

- 무대는 `아프리카 밀림`이다.
- 밀렵꾼들이 작은 동물부터 큰 동물까지 붙잡아 두고 있다.
- 플레이어는 퍼즐을 통해 우리와 덫을 정리하고 동물들을 풀어 준다.
- 초반은 작은 동물 구조, 중반은 야영지와 운반 경로 정리, 후반은 코끼리 구출 준비, 최종전은 `거대한 코끼리 구출`로 이어진다.
- 자세한 밴드별 스토리는 [story-campaign.md](/Users/shinehandmac/Documents/puzzle/docs/game/story-campaign.md)를 따른다.

## 4. 코어 게임 루프

1. 스테이지를 선택한다.
2. 시작 목표와 제한 이동 수를 확인한다.
3. 동물 블록을 스와이프해 3개 이상 매치한다.
4. 연쇄 매치와 특수 효과로 목표를 달성한다.
5. 클리어 보상 또는 재도전을 선택한다.

## 5. 퍼즐 규칙

### 기본 규칙

- 블록은 2개를 교환할 수 있다.
- 교환 결과가 3개 이상 매치일 때만 이동이 성립한다.
- 매치는 가로 또는 세로 기준이다.
- 매치된 블록은 제거되고 위 블록이 아래로 떨어진다.
- 새 블록은 위에서 채워진다.
- 추가 매치가 생기면 자동으로 연쇄 처리한다.

### 패배 조건

- 제한 이동 수를 모두 사용했는데 목표를 달성하지 못한 경우

### 승리 조건

- 스테이지별 목표를 모두 달성한 경우

## 6. 블록 콘셉트

### 기본 동물 블록 5종

- 토끼: 밝고 빠른 느낌의 대표 캐릭터
- 곰: 둥글고 안정적인 형태
- 고양이: 표정 변화가 잘 드러나는 캐릭터
- 병아리: 작은 크기와 선명한 색상 포인트
- 개구리: 실루엣이 뚜렷한 변형형 캐릭터

### 표현 원칙

- 실루엣만 봐도 종류가 구분되어야 한다.
- 작은 화면에서도 눈, 귀, 입 형태가 읽혀야 한다.
- 색상과 얼굴형을 동시에 구분 요소로 사용한다.

## 7. 스테이지 목표 구조

### MVP 목표 타입

- 특정 동물 블록 N개 수집
- 점수 N점 달성
- 특정 턴 안에 목표 수집 완료

### 스토리 해석

- 동물 수집 목표는 `잡힌 동물을 풀어 준 수`
- 점수 목표는 `구조 속도와 구조 효율`
- 장애물 목표는 `덫, 시야 방해, 지형 장벽 제거`
- 복합 목표는 `한 구역의 구조 임무 전체 수행`

### 추후 확장 목표

- 장애물 제거
- 바닥 타일 색칠
- 특정 구역 해방

### 현재 포함 장애물

- 덤불: 매치된 칸 또는 인접 칸의 충격으로 제거된다.
- 덤불은 중반 이후 스테이지의 주요 목표로 사용한다.

## 8. 특수 블록

### MVP 판단

- 현재 버전부터 `핵심 재미 요소`로 포함
- 우선순위는 기본 매치 감각 다음이며, 연쇄와 함께 게임성을 만드는 핵심 장치다.

### 포함 시 우선 후보

- 4개 매치: 가로 또는 세로 줄 제거 블록
- 5개 매치: 범위 폭발 또는 동일 종류 제거 블록

### 현재 확정 규칙

- 가로 4매치: `가로 줄 제거` 특수 블록 생성
- 세로 4매치: `세로 줄 제거` 특수 블록 생성
- 5개 이상 매치: `3x3 폭발` 특수 블록 생성
- 특수 블록이 매치에 포함되거나 다른 특수 효과에 닿으면 즉시 발동한다.
- 특수 블록 생성 위치는 가능한 경우 `사용자가 마지막에 스왑한 칸`을 우선한다.

## 9. HUD 및 화면 구성

### 플레이 화면 필수 정보

- 현재 스테이지 번호
- 현재 구조 임무의 짧은 맥락
- 남은 이동 수
- 현재 목표 진행도
- 일시정지 버튼

### landscape 기준 배치 방향

- 중앙 또는 좌측 중심: 퍼즐 보드
- 우측 패널: 목표, 이동 수, 점수
- 상단: 스테이지 정보와 일시정지

## 10. 피드백 원칙

- 유효 스왑: 즉시 이동 + 짧은 탄성 피드백
- 무효 스왑: 원위치 복귀 + 약한 흔들림
- 매치 성공: 동물 표정 변화, 팝 이펙트, 점수 상승, 구조 진행 강조
- 연쇄 성공: 화면 강조, 추가 사운드, 콤보 텍스트
- 클리어 직전: 목표 영역 강조, `구출 거의 완료` 느낌 강화

## 11. 초기 스테이지 곡선

### Stage 1

- 목표: 토끼 10개 수집
- 이동 수: 12
- 목적: 기본 스와이프와 매치 이해
- 스토리: 정글 입구에 갇힌 토끼들을 먼저 구조

### Stage 2

- 목표: 병아리 10개 + 토끼 6개 수집
- 이동 수: 13
- 목적: 목표 칩을 보며 우선순위 판단
- 스토리: 작은 우리 2곳을 동시에 열어 구조 동선을 넓힘

### Stage 3

- 목표: 곰 8개 + 고양이 8개 수집
- 이동 수: 14
- 목적: 복수 목표와 4매치 특수 블록 학습
- 스토리: 더 단단한 묶음을 풀기 위해 강한 구조 액션 학습

### Stage 4

- 목표: 개구리 10개 + 점수 2500점 + 덤불 6개 제거
- 이동 수: 15
- 목적: 점수와 수집 목표 동시 관리
- 스토리: 덤불 뒤에 숨겨진 덫과 길목 정리

### Stage 5

- 목표: 점수 3000점 + 덤불 8개 제거
- 이동 수: 15
- 목적: 5매치 폭발 블록과 연쇄 가치 학습
- 스토리: 구조 통로를 크게 터서 다음 구역 진입

### Stage 6

- 목표: 토끼 8개 + 곰 8개 + 개구리 8개 + 점수 3500점 + 덤불 10개 제거
- 이동 수: 16
- 목적: 복합 목표 하드 스테이지 검증
- 스토리: 여러 동물을 동시에 풀며 첫 구역 구조 완료

## 12. 캠페인 스토리 구조

- 100스테이지는 10개 밴드로 나뉜다.
- 각 밴드는 하나의 정글 구역과 구조 임무를 가진다.
- 각 밴드의 마지막 스테이지는 미니 피날레로 다룬다.
- 최종 목표는 `거대한 코끼리 구출`
