# Alpha Approval Criteria

## 문서 목적

이 문서는 `Animal Pop Match`가 알파 품질로 인정받기 위한 **최소 통과 기준**을 단일 문서로 잠근다.
팀별 기준 충돌이 발생하면 이 문서를 최우선 기준으로 삼는다.

---

## 알파 단계 정의

- 단계명: `제품화 알파`
- 목표 상태: `프로토타입 UI + 코어 루프`에서 `제품으로 느껴지는 첫 경험` 수준으로 올라온 상태
- 포함: 기능 품질, 학습 설계, 시각 가독성
- 제외: 라이브 운영, 과금/광고, BGM, 신규 장애물 확장

---

## 통과 기준 (AND 조건)

### UX 기준

| 기준 | 판단 방법 |
|------|-----------|
| 홈에서 첫 플레이까지 설명 없이 이해 가능 | 첫 방문자가 `시작` 버튼 누름 → Stage 선택 → 인게임 진입까지 안내 없이 수행 가능 |
| 첫 10초 안에 목표·이동 수·첫 행동이 인지됨 | Stage 1 시작 직후 첫 매치까지 혼란 없음 |
| 플레이 화면에서 개발용 UI 비노출 | ShuffleButton, DemoMatchButton 등 개발 전용 노드 없음 |
| 클리어/실패 오버레이가 다음 행동을 명확히 제시 | 클리어: 다음 스테이지 또는 홈. 실패: 재도전 또는 홈 |

### 학습 기준

| 기준 | 판단 방법 |
|------|-----------|
| Stage 1 소프트 튜토리얼 존재 | Stage 1 진입 시 첫 안내 배너가 표시됨 |
| 밴드 전환 지점(11·25·45·65·85·95)에 학습 장치 존재 | 각 지점 진입 시 해당 스테이지 `tutorial` 필드가 비어 있지 않음 |
| 튜토리얼 1회성 저장 동작 | 같은 스테이지 재진입 시 튜토리얼 배너 미노출 |

### 기술 기준

| 기준 | 판단 방법 |
|------|-----------|
| `validate_gameplay.sh` 통과 | 스테이지 JSON 구조·밸런스 검증, 헤드리스 로드 오류 없음 |
| portrait/landscape 레이아웃 정상 동작 | 두 방향 모두 보드와 HUD가 겹치지 않음 |
| Android 실기기 사운드 정상 재생 | 탭·스왑·매치 사운드가 기기에서 들림 |
| Android 실기기 햅틱 정상 동작 | VIBRATE 권한 반영, 탭·스왑 진동이 기기에서 느껴짐 |

## 반려 조건 (OR 조건)

아래 중 하나라도 발생하면 QA가 반려한다.

1. 첫 10초 안에 목표·이동 수·첫 행동이 이해되지 않음
2. 개발용 UI(버튼·레이블)가 한 군데라도 노출됨
3. HUD가 보드 가독성을 침범함
4. 특수 블록·덤불·선택 상태가 즉시 구분되지 않음
5. Stage 11·25·45·65·85·95 중 학습 지점이 비어 있음
6. `validate_gameplay.sh`가 실패함

---

## 대표 검증 코스

수동 QA는 아래 순서로 진행한다.

1. 홈 화면 (`시작`, `스테이지 라인`, `설정` 동작 확인)
2. Stage 1 (튜토리얼, 첫 매치, 클리어 오버레이)
3. Stage 11 (학습 배너, 밴드 전환 인지)
4. Stage 25 (비대칭 보드, 학습 지점)
5. Stage 50 (중반 압박, 별 등급 계산)
6. Stage 75 (탈출 준비 구간, 복합 목표)
7. Stage 100 (피날레, 전체 클리어 오버레이)

---

## 승인 권한

- **QA Lead**가 위 반려 조건 없음을 확인 후 `알파 통과` 판정을 내린다.
- **PM Lead**가 대표 검증 코스 통과를 최종 확인 후 릴리즈 준비 단계로 넘긴다.

---

## 이 문서의 갱신 규칙

- 승인 기준을 완화하거나 추가할 때는 PM Lead와 QA Lead가 동의해야 한다.
- 구현 현황에 따라 `통과/반려` 판단 방법만 갱신할 수 있다.
