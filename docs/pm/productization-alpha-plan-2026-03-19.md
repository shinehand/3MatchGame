# Productization Alpha Plan

## 현재 판단

- 현재 프로젝트 단계는 `게임 코어 + 100스테이지 데이터 + 프로토타입 UI`다.
- 이번 라운드는 `정식 출시 준비`가 아니라 `제품화 알파`로 정의한다.
- 핵심 목표는 기능 추가보다 `기획 기준선 정리`, `아트 제품화`, `학습 품질 개선`, `UI 품질 상승`이다.

## 이번 라운드 우선순위

1. 아트 고도화
2. 기획 기준선 정리
3. UI 제품화 반영
4. QA 게이트 강화
5. 후반 확장 입력 잠금

## 팀별 책임

- PM Team
- 스코프, 승인 기준, 제외 범위 고정
- Planning Team
- 문서 충돌 제거, 튜토리얼/밸런스 기준 잠금
- Art Team
- UI 테마 팩과 자산 패키지 정의 및 1차 산출
- Development Team
- 설정/튜토리얼/UI 자산 연결
- QA Team
- 대표 스테이지 검증과 릴리즈 차단 조건 정의

## 승인 기준

- 홈에서 첫 플레이까지 설명 없이 이해 가능
- Stage 1과 밴드 전환 학습 지점에 소프트 튜토리얼 존재
- 플레이 화면에서 개발용 UI 비노출
- UI 테마 팩 1차 반영 후 정보 위계가 `보드 > 목표 > 이동 수 > 다음 행동`으로 읽힘
- 자동 검증과 대표 스테이지 수동 검증 통과

## 2주 실행 순서

### 1주차

- PM
- 작업 브리프와 승인 기준 잠금
- Planning
- 기준선 문서 정리
- Art
- 이미지 생성 전략 결정
- UI 테마 팩, 보드 상태 세트, 메타 UI 세트 1차 정의

### 2주차

- Development
- 튜토리얼/설정/UI 자산 반영
- QA
- 대표 스테이지 검증
- PM
- 승인/보류 결정

## 리스크 관리

- 문서와 구현의 시간차가 크므로 `현재 구현 기준 문서`를 우선 기준으로 본다.
- 아트 산출은 한 번에 전량 제작하지 않고 `패키지 단위`로 나눠 개발에 넘긴다.
- 밴드 배경은 1-20만 먼저 잠그고 이후 확장한다.
