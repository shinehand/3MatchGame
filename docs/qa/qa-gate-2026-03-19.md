# QA Gate

## 현재 QA 판단

- 현재 단계는 `코어 플레이 검증 가능`, `제품 품질 승인 불가`다.

## 이번 라운드 핵심 차단 조건

1. 첫 10초 안에 목표, 이동 수, 첫 행동이 이해되지 않으면 반려
2. 개발용 UI가 한 군데라도 노출되면 반려
3. HUD가 보드 가독성을 침범하면 반려
4. 특수 블록/덤불/선택 상태가 즉시 구분되지 않으면 반려
5. Stage 11, 25, 45, 65, 85, 95 중 학습 지점이 비어 있으면 반려

## 대표 검증 코스

- 홈
- Stage 1
- Stage 11
- Stage 25
- Stage 50
- Stage 75
- Stage 100

## 체크리스트

- 홈에서 `이어하기`, `플레이`, `스테이지 선택` 우선순위가 명확한지
- Stage 1 시작 직후 첫 매치까지 혼란이 없는지
- 밴드 전환 튜토리얼이 실제 플레이를 방해하지 않고 방향만 제시하는지
- portrait 레이아웃에서 `홈`, `재시작`, `일시정지`, `다음` 의미가 명확한지
- 클리어/실패 오버레이가 다음 행동을 명확히 제시하는지
- 저장/해금/별 수가 대표 스테이지 전환에서 정상인지
- Android 실기기에서 사운드와 햅틱 동작이 안정적인지

## QA Team 역할

- Feature QA
- 요구사항 충족 여부 검증
- UX QA
- 가독성, 상태 인지, 온보딩 검증
- Device QA
- Android 실기기 확인
- Regression QA
- 자산 반영 후 기존 코어 루프 회귀 확인
